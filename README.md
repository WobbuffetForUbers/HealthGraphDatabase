# 🏥 Clinical QI Copilot

**Clinical QI Copilot** is an AI-driven analytical engine designed to bridge the gap between complex clinical data and actionable hospital quality improvement. It combines a high-fidelity **Neo4j Knowledge Graph** with an **OpenRAG architecture** to provide grounded, protocol-aware insights into patient flow, hospital capacity, and clinical outcomes.

---

## 🌟 Key Features

### 1. Clinical Knowledge Graph (Neo4j)
- **High-Fidelity Schema**: Maps relationships between `Patients`, `Encounters`, `Providers`, `Conditions`, `Medications`, and `Observations`.
- **Multi-Hop Bottleneck Logic**: Native Cypher generation for analyzing transitions between care settings (e.g., ED-to-Inpatient "Boarding Times").
- **Automated ETL**: Custom pipeline (`etl_synthea.py`) for ingesting and linking large-scale clinical datasets.

### 2. OpenRAG Architecture (OpenSearch + Gemini)
- **Protocol Grounding**: Automatically searches the **UCSF Hospitalist & Outpatient Handbooks** to ensure data analysis aligns with established clinical guidelines.
- **Vector Search**: Utilizes OpenSearch 3.5.0 with the `faiss` k-NN engine and `models/text-embedding-004` for high-performance similarity search.
- **Robust Ingestion**: Includes a headless browser scraper (Playwright) to index medical resources and local PDF policies.

### 3. Intelligent Visualization Engine
- **AI-Driven Charts**: Automatically selects the most impactful visualization (Bar, Line, Area, or Scatter) based on the clinical question and data structure.
- **Clinical Interpretation**: Provides concise findings summaries and rationales for every data trend identified.
- **Insights Archive**: Persistently stores analytical findings in a dedicated sidebar for team review.

### 4. Model Context Protocol (MCP)
- **Standardized Access**: Exposes clinical tools (`run_cypher`, `search_guidelines`) via a FastMCP server, allowing any MCP-compatible LLM to interact with the database.

## 🚀 Getting Started

### 1. Prerequisites
- Docker & Docker Compose
- Python 3.10+
- A Gemini API Key from [Google AI Studio](https://aistudio.google.com/app/apikey)

### 2. Installation
```bash
# Clone the repository
git clone https://github.com/WobbuffetForUbers/HealthGraphDatabase.git
cd HealthGraphDatabase

# Install dependencies
pip install -r requirements.txt

# Setup environment variables
# Create a .env file and add:
GEMINI_API_KEY="your_api_key_here"
```

### 3. Launch Infrastructure
```bash
docker-compose up -d
```

### 4. Ingest Data
```bash
# Load clinical CSVs into Neo4j
python etl_synthea.py

# (Optional) Run the web scraper to index clinical guidelines
python tools/web_ingest.py
```

### 5. Run the Copilot
```bash
streamlit run app.py
```

## 🔒 Security & Standards
- **API Integrity**: All credentials managed via `.env` and strictly excluded from source control.
- **Native Cypher**: Forbids APOC in favor of native Cypher functions for maximum stability.
- **Temporal Handling**: Mandates `datetime()` casting for all clinical timestamps to ensure accurate throughput calculations.
